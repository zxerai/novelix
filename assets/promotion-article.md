# 我开源了一个 AI 写小说的 Agent 系统，它能 24 小时不停写完整本网文

写网文最痛苦的是什么？

不是没灵感，不是不知道怎么开头——是**写完 10 章之后，前面埋的伏笔忘了收，角色的实力忽高忽低，上章丢掉的武器下章又出现了**。

我花了几个月做了一个叫 **Novelix** 的东西，本质上是一个由 10 个 AI Agent 组成的"写小说流水线"。

## 它能做什么？

```bash
# 装
npm i -g @actalk/novelix

# 创建一本书
novelix book create --title "吞天魔帝" --genre xuanhuan

# 写下一章 — 全自动：构思 → 写作 → 审计 → 修订
novelix write next 吞天魔帝

# 导出
novelix export 吞天魔帝 --format epub
```

你不用盯着它写。它自己会写、自己审、自己改。一章写完自动跑下一章，你可以关机走人，第二天回来看结果。

## 它不是"让 ChatGPT 写小说"

如果你用过 ChatGPT/Claude 直接写小说，应该遇到过这些问题：

- 写了 5 章之后忘了前面的设定
- 主角的名字中途变了
- 人物的实力忽上忽下
- 前面埋的伏笔再也没提过
- "仿佛""不禁""宛如"高频重复，AI 味太重

Novelix 解决这些问题的方式不是"写更好的 prompt"，而是一个 **10 Agent 接力管线**：

| Agent | 干的事 |
|-------|--------|
| **雷达** | 扫描平台趋势，告诉写手现在读者喜欢什么 |
| **规划师** | 读作者意图 + 当前焦点，决定本章写什么 |
| **编排师** | 从全量真相文件中精选上下文，避免上下文爆炸 |
| **建筑师** | 建书时生成完整世界观、规则、角色 |
| **写手** | 基于编排后的精简上下文写正文 |
| **观察者** | 提取 9 类事实（角色/位置/资源/关系等） |
| **反射器** | 输出 JSON delta，代码层做 schema 校验 |
| **归一化器** | 字数超了就压缩，不够就补足 |
| **审计员** | 对照 7 个真相文件做 33 维度连续性检查 |
| **修订者** | 修复审计发现的问题，修完再审一次 |

每个 Agent 各司其职，像工厂流水线一样。

## 核心卖点：连续性

Novelix 每本书维护 **7 个真相文件**作为唯一事实来源：

- `current_state.md` — 角色位置、关系、情感状态
- `particle_ledger.md` — 每件物品的数量和位置
- `pending_hooks.md` — 埋了哪些伏笔、哪些还没收
- `chapter_summaries.md` — 每章干了什么
- `subplot_board.md` — A/B/C 支线走到哪了
- `emotional_arcs.md` — 每个角色的情绪变化
- `character_matrix.md` — 谁认识谁、知道什么

审计员每章对照这些文件检查。如果主角"想起"了从未见过的事，或者拿出两章前已经丢掉的武器，审计员会当场抓出来。

## 去 AI 味

大模型写的文字很容易被朱雀、Originality.ai 这类工具检测出来。Novelix 内置了一套反检测改写策略，涵盖了 **22 条具体规则**，包含：

- **句式变异**：长短句剧烈交替，不得连续 3 句以上同长度
- **词汇替换**：消灭"仿佛、不禁、宛如、突然、忽然、瞬间、顿时"等 AI 高频词
- **段落呼吸**：段落长度陡峭变化，有的段只有一句话，有的段七八行
- **情绪外化**：不写"他感到愤怒"，写"他捏碎了茶杯，茶水从指缝往下滴"
- **朱雀专项**：句长波动、标点多样性、代词降频、避免完美过渡

```bash
# 对已有章节执行反检测改写
novelix revise 吞天魔帝 3 --mode anti-detect
```

## 实测数据

用 Novelix 全自动跑了一本玄幻《吞天魔帝》：

- 31 章
- 45 万字
- 平均每章 ~14,500 字
- 审计通过率 100%
- 零人工干预

## 不只是命令行

如果你不习惯终端操作，Novelix 也附带了一个 Web 工作台：

```bash
novelix
# 打开 http://localhost:4567
```

在浏览器里可以：管理书籍、审阅章节、看字数趋势图、查数据分析、看角色关系图谱、配模型、导出——CLI 能做的，Web 上都能做。

## 谁适合用？

- **网文写手**：想用 AI 辅助创作，但受不了 ChatGPT 丢三落四
- **技术爱好者**：对多 Agent 系统、LLM 应用感兴趣
- **同人创作者**：内置同人模式，支持从原文导入正典
- **需要量产内容的人**：短篇、番外、仿写支持

## 快速上手

```bash
# 安装
npm i -g @actalk/novelix

# 初始化项目
novelix init my-novel

# 配模型（首次运行会自动引导）
novelix doctor

# 创建一本书
novelix book create --title "我的第一本书" --genre xianxia

# 全自动写
novelix write next 我的第一本书 --count 10
```
